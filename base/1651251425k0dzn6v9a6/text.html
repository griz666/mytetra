"Вместо этого вы можете заставить Docker автоматически вращать журналы для вас. Это делается с дополнительными флагами для dockerd, если вы используете драйвер ведения журнала JSON по умолчанию:
dockerd ... --log-opt max-size=10m --log-opt max-file=3

Вы также можете установить это как часть вашего файла daemon.json вместо изменения сценариев запуска:

{
  "log-driver": "json-file",
  "log-opts": {"max-size": "10m", "max-file": "3"}
}

Эти параметры должны быть настроены с корневым доступом. Обязательно запустите a systemctl reload dockerпосле изменения этого файла, чтобы применить настройки. Этот параметр будет использоваться по умолчанию для всех вновь созданных контейнеров. Обратите внимание, что существующие контейнеры необходимо удалить и воссоздать, чтобы получить новые ограничения журнала.

Аналогичные параметры журнала могут быть переданы отдельным контейнерам, чтобы переопределить эти значения по умолчанию, позволяя сохранять больше или меньше журналов в отдельных контейнерах. Из docker runэтого следует, что:

docker run --log-driver json-file --log-opt max-size=10m --log-opt max-file=3 ...

или в файле compose:

version: '3.7'
services:
  app:
    image: ...
    logging:
      options:
        max-size: "10m"
        max-file: "3"

Для дополнительной экономии места вы можете переключиться с драйвера журнала json на "локальный" драйвер журнала. Он использует те же параметры max-size и max-file, но вместо хранения в json он использует двоичный синтаксис, который быстрее и меньше. Это позволяет хранить больше журналов в файле того же размера. Демон.запись json для этого выглядит так:

{
  "log-driver": "local",
  "log-opts": {"max-size": "10m", "max-file": "3"}
}

Недостатком локального драйвера являются внешние парсеры / форвардеры журналов, которые зависели от прямого доступа к журналам json, больше не будут работать. Поэтому, если вы используете такой инструмент, как filebeat, для отправки в Elastic или универсальный форвардер Splunk, я бы избегал "локального" драйвера.

Я немного подробнее расскажу об этом в своей презентации Tips and Tricks ."
